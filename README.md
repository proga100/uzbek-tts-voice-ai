# Uzbek TTS Fine-tune — Research Proof

**Goal:** prove that fine-tuning a custom Uzbek text-to-speech voice is feasible, by
fine-tuning an open VITS model on the clean single-speaker **FeruzaSpeech** corpus and
generating Uzbek audio in that voice.

This is a **research proof-of-concept**, kept fully separate from the Yandex voice-agent
codebase. It does **not** touch `../yandex`.

## ⚠️ Licensing — read before using output anywhere

This pipeline is for **research / demonstration only**, NOT for a commercial product:

- **FeruzaSpeech** — "freely available for **academic research** purposes" only.

So the resulting voice **cannot ship commercially**. It exists only to demonstrate the training

## What it does

```
FeruzaSpeech (clean, 1 female speaker, 16 kHz, Latin transcripts)
  → fine-tune facebook/mms-tts-uzb-script_latin (VITS) via ylacombe/finetune-hf-vits
  → generate an Uzbek sample WAV in the fine-tuned voice
```

VITS/MMS is lightweight enough to fine-tune on Kaggle's free GPU, and the recipe reports good
results in ~20 minutes with as few as 80–150 samples — ideal for a fast proof.

## Run it on Kaggle

1. Create a free **Kaggle** account → New Notebook → **Settings → Accelerator → GPU (T4 x2)**.
2. Create a **Hugging Face** account, then:
   - Visit https://huggingface.co/datasets/k2speech/FeruzaSpeech and **accept the terms** (gated).
   - Create a token at https://huggingface.co/settings/tokens (read access).
3. In the Kaggle notebook: **Add-ons → Secrets** → add a secret named `HF_TOKEN` with your token.
4. Upload `feruzaspeech_mms_finetune.ipynb` (File → Import Notebook) and **Run All**.
5. Output: a fine-tuned model in `/kaggle/working/mms-uzb-feruza/` and `sample.wav` to listen to.

> Note: exact dataset column names / convert-script flags may need a one-line tweak the first
> run — they're marked `# ADJUST` in the notebook. This scaffold gets you 95% there; we iterate
> from the first Kaggle run.

## Path to production (commercial voice)

The proof shows the pipeline works. For a **shippable** voice you need clean **and**
commercially-licensed data + a permissive base model:

1. **Record your own narrator** — ~3–10 h, one Uzbek voice actor, studio-clean, with a
   **rights buyout** in the contract. You own the voice outright.
2. **Base model: Piper (VITS, MIT)** — commercial-safe (instead of MMS/CC-BY-NC).
3. Train (same idea as this notebook), evaluate vs the current Yandex `nigora`, then serve
   behind the app's TTS provider interface.
